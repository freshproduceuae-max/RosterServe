RS-F008 NOTABLE IMPLEMENTATION DECISIONS (carry-forward reference)
- assignments table: one active assignment per volunteer per event per department (partial unique index); sub_team_id is placement-only, not a slot dimension
- 8 RLS policies: dept_head (SELECT/INSERT/UPDATE), sub_leader (SELECT/INSERT/UPDATE), super_admin SELECT, volunteer SELECT
- Sub-leader scope: write within owned sub-teams only; sub_team_id IS NOT NULL enforced; role IN ('volunteer','sub_leader') enforced; department_id check on sub_team added to prevent cross-dept invariant violations
- canViewRoster prop in department-detail-card: isSubLeaderInDept = sub_leader with at least one owned sub-team in this dept (separate from canManage)
- getVolunteersForAssignment: multi-step JS merge approach (events → volunteer_interests → blockouts → skills → assignments); availability is advisory (amber chip, not a blocker)
- AssignmentRow vs AssignmentCard: separate components for desktop table (td cells) and mobile card; shared state via useAssignmentRowState hook
- requireSubTeam prop threaded through AssignmentList → AssignmentRow/AssignmentCard: sub-leader views pass true, dept_head false; removes "No sub-team" escape hatch from sub-leader edit UI
