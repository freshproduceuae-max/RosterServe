- UPDATE policies use explicit USING + WITH CHECK (RS-F006 pattern):
  - Dept_head review: USING gates on pending + role + ownership; WITH CHECK gates status IN ('approved','rejected') + reviewed_by = auth.uid() + reviewed_at IS NOT NULL
  - Volunteer withdraw: USING gates on pending + not-deleted; WITH CHECK gates deleted_at to 30-second window + reviewed_by IS NULL + reviewed_at IS NULL
- Legacy onboarding skills (NULL department_id, NULL skill_id): visible in volunteer view with muted label and withdraw control; absent from all leader views
- Soft-deleted catalog skills: FK on volunteer_skills.skill_id is NOT nulled (soft-delete doesn't cascade); claims remain valid using stored name column; skill disappears from claim form only
